# 验收清单 & 证据规格（Acceptance Checklist + Evidence Spec）

> 本文件将“验收口径”与“证据要求”固化，作为索引DONE判定依据。
> 证据命名/目录结构/索引字段：遵循 `evidence/LOCK.md`。

## A. 通用DONE标准（适用于所有条目）
每条验收/证据达到 DONE 时，必须满足：
1) **一跳定位**：从 `evidence/00-index.md` 可直接定位到证据文件
2) **可复核**：证据内版本号/时间点与索引登记一致
3) **可追溯**：至少具备 关联指针（决议 → 行动项 → 风险），或明确说明不适用

## B. T3-Gate（本阶段）
### B1. 硬证据#1：可签字评审决议包（evidence/01_review/）
- 必备文件：
  - 评审记录（review-record）
  - 反馈清单（feedback-list）
  - 决议项（decision-items，可签字）
  - 签字页/书面确认（pdf/png/邮件截图）
  - 一致性校验记录（版本号/时间点与索引字段一致）
- 验收：上述文件齐备且可执行、可签字；索引条目 DONE 且附验证记录。

### B2. 硬证据#2：VPS采购确认证据包（evidence/02_procurement/）
- 必备文件：
  - 确认单/订单号
  - 交付时间来源原始证据（邮件/工单/控制台截图等）
  - 规格/地域说明
  - 付款凭证或等效证明
- 验收：信息完整可复核；交付时间可追溯；索引条目 DONE 且附验证记录。

### B3. 硬证据#3：端到端归档演练报告（evidence/09_archiving_drill/）
- 必备内容：
  - 覆盖硬证据#1/#2全链路：采集→命名→关联→索引更新→TODO→DONE→可追溯性验证
  - 问题清单与修复闭环（至少闭环到可追溯达标）
- 验收：报告可复现；索引更新过程可证明；索引条目 DONE 且附验证记录。
