# 技术栈选型决策文档

## 后端语言对比

| 维度 | Go | Python | Node.js |
|------|----|---------|----|
| 开发效率 | 中 | 高 | 高 |
| 性能 | 优 | 中 | 中 |
| 部署复杂度 | 低（单二进制） | 中（需运行时） | 中（需Node运行时） |
| 学习成本 | 中 | 低 | 低 |
| 生态（CI/CD） | 成熟 | 成熟 | 成熟 |
| 推荐指数 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

**推荐：Go**
- 理由：单二进制部署，性能优，适合一人团队快速迭代
- Trade-off：学习曲线略陡，但长期收益高

## 基础设施方案对比

| 维度 | 云服务（AWS/阿里云） | 自建（VPS+Docker） |
|------|-----|-----|
| 初期成本 | 高（按量计费） | 低（固定VPS费用） |
| 运维负担 | 低 | 中 |
| 扩展性 | 优 | 需手动扩展 |
| 可靠性 | 高（SLA保证） | 中（依赖VPS提供商） |
| 学习成本 | 高 | 中 |
| 推荐指数 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

**推荐：自建（VPS+Docker）**
- 理由：成本可控，一人团队可维护，快速启动
- Trade-off：需要基础的Docker和Linux运维知识

## CI/CD方案对比

| 方案 | 功能完整性 | 集成难度 | 成本 | 推荐指数 |
|------|--------|--------|------|--------|
| GitHub Actions | 高 | 低 | 免费（私有仓库有限制） | ⭐⭐⭐⭐⭐ |
| GitLab CI | 高 | 低 | 免费（自建）或付费 | ⭐⭐⭐⭐ |
| 自研 | 定制化 | 高 | 开发成本高 | ⭐⭐ |

**推荐：GitHub Actions**
- 理由：零成本，与GitHub无缝集成，功能完整，一人团队最优选择
- Trade-off：依赖GitHub平台

## 最终技术栈决策

```
后端：Go 1.21+
基础设施：单VPS（2核4GB起）+ Docker + Docker Compose
CI/CD：GitHub Actions
数据库：PostgreSQL 14+
监控：Prometheus + Grafana（可选，MVP阶段可用简单日志）
日志：ELK Stack（可选，MVP阶段可用文件日志）
```
